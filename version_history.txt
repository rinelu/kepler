Kepler – Revision History
================================
Semantic versioning: MAJOR.MINOR.PATCH

--------------------------------
v0.13.0 – Atmospheres & Renderer Refactor
--------------------------------

- Added atmospheric rendering pass for celestial bodies
- Refactored body renderer into begin/draw/end stages
- Move RenderContext to its own file
- Improved LOD selection using projected screen size
- Rewrote camera system for stability and smoother controls
- Cleaned up world iteration and body lifetime handling
- General renderer, shader, and math cleanup

Notes:
- Atmospheres are view-based only (no shadows/eclipses yet)

------------------------------------------------
v0.12.0 – Prediction & Editor Improvements
------------------------------------------------

Changes
- Refactored prediction system to use contiguous buffers and stride-based sampling
- Improved prediction rendering using rlgl line batching
- Prediction will only rebuilds when bodies, world, or prediction settings change
- Exposed prediction timestep (`dt`) and improved parameter clamping
- Minor editor, camera, and GUI cleanup

Removed
- Legacy per-body prediction track allocations
- Unused raylib-nuklear CMake include target

------------------------------------------------
v0.11.0 – Engine Restructure & Scheduler Refactor
------------------------------------------------

Changes
- Migrated Dear ImGui to source-based build
- Migrated rlImGui to source-based build
- Major source tree reorganization for clearer module boundaries:
    - Split former `core` into `engine`, `platform`, and `predict`
    - Moved scheduler and engine code under `engine`
    - Isolated platform-specific systems (input, time)
    - Grouped render passes under `render/passes`
    - Reorganized GUI layers by backend (`imgui`, `nuklear`)
    - Relocated editor-related code under `editor`
    - Introduced `kepler` module as primary application entry point
- Scheduler refactor:
    - Removed dependency on `dynamic_array`
    - Replaced macro-based container logic with explicit dynamic storage
    - Preserved deterministic system execution order
- Engine and system cleanup:
    - Updated include paths to reflect new module layout
    - Improved internal consistency across engine, render, and system layers

Removed
- Legacy `dynamic_array` utility
- Deprecated `app` interface and headers

------------------------------------------------
v0.10.0 – Dual GUI Backends, CI, and Platform Unification
------------------------------------------------

Changes
- Added Nuklear GUI backend alongside existing ImGui integration:
    - New C-based Nuklear layer via raylib-nuklear
    - Shared GUI entry point renamed (imgui_layer -> gui_layer)
- GUI and interaction refactors:
    - Centralized GUI input capture handling across backends
    - Improved selection overlay rendering consistency
    - Cleaner separation between world rendering and UI overlays
- Introduced GitHub Actions CI pipeline:
    - macOS, Linux (gcc & clang), and Windows (MSYS2)
    - Ensures cross-platform build stability
- Reworked third-party raylib layout:
    - Consolidated headers under thirdparty/raylib/include
    - Platform-specific binaries organized by target (linux / macos / win64 / wasm)
    - Removed redundant upstream documentation files
- Engine, camera, and input improvements:
    - Cleaner GUI-aware input routing
    - Minor camera handling fixes
    - Renderer stability improvements
- General cleanup:
    - Utility and renderer header refactors
    - Selection overlay fixes

Known Limitations
- Nuklear UI does not yet reach feature parity with ImGui panels
- No runtime GUI backend switching (compile-time / manual selection)
- Inspector edits remain non-persistent and lack undo/redo

------------------------------------------------
v0.9.0 – Interaction, Selection, and Editor Tooling
------------------------------------------------

Changes
- Added mouse-based body picking using camera ray casting
- Introduced persistent body selection state in Engine
- Visual selection indicator rendered in-world (billboarded animated ring)
- On-screen selection label overlay for selected bodies
- Added Inspector panel for live per-body editing:
    - Transform (position, velocity, acceleration)
    - Physics parameters (mass, radius, damping)
    - Rendering flags and material properties
    - Light / emissive tuning with blackbody temperature control
- Centralized input handling system:
    - Unified keyboard and mouse input processing
    - Proper GUI input capture handling (ImGui-safe)
- Camera system improvements:
    - GUI-aware mouse and keyboard handling
    - Runtime camera mode switching (free / orbit / follow)
    - Follow target selection via GUI
    - Orbit camera panning with middle mouse
    - Smooth interpolation of orbit/follow parameters
    - Exposed camera parameters in editor: Sensitivity, Move speed, Zoom speed, Distance / yaw / pitch
    - Improved FREE camera interaction reliability
- Refactored shared math and color utilities into utils module
- Moved global render resources (default meshes/shaders) into core globals
- Improved star rendering:
    - Temperature-driven base color assignment
    - Revised emissive contribution scaling
- Tuned bloom extraction threshold for better HDR response
- Improved visual radius scaling for large-mass bodies

Known Limitations
- Barnes–Hut gravity still violates Newton’s 3rd law
- No collision detection or body merging
- No floating-origin or large-scale precision correction
- Selection overlay does not account for occlusion by UI
- Inspector edits are not persisted or undoable

------------------------------------------------
v0.8.0 – Rendering, Camera, and Physics Enhancements
------------------------------------------------

Changes
- Full rewrite of celestial rendering using BodyRender abstraction
- Added bloom post-processing with HDR extraction, blur, and combine passes
- Added glow/rim shaders for emissive stars
- Multi-LOD sphere meshes for planets (near, mid, far)
- Unified camera system:
    - Free, orbit, and follow modes fully integrated
    - Smooth interpolation for position, yaw, pitch, and distance
    - Mouse axis inversion configurable via GUI
- Barnes–Hut gravity improvements and bug fixes (`bh_compute_gravity`)
- Adaptive physics timestep scaling with sub-stepping for high-acceleration bodies
- Fixed planet shader normals
- Optimized render data and body creation pipeline

Known Limitations
- Barnes–Hut still violates Newton’s 3rd law
- No collision or body merging support
- Floating-origin support not yet implemented
- Some visual effects (atmospheres) are incomplete

------------------------------------------------
v0.7.0 – Lighting, Orbits, and Physics Stability
------------------------------------------------

Changes
- Multi-light point lighting with distance attenuation
- Blackbody temperature → RGB for star coloration
- Emissive stars and atmospheric rim lighting
- Explicit orbital initializers (Keplerian, virial rotating)
- Adaptive physics sub-stepping based on max acceleration
- Automatic switch between N² gravity and Barnes–Hut
- Simplified body creation and render data cleanup

Known Limitations
- Barnes–Hut violates Newton’s 3rd law
- No collisions or body merging
- No floating-origin support

--------------------------------
v0.6.0 – Diagnostics & Introspection
--------------------------------

Changes
- Total energy (K + U) tracking per fixed step
- Linear and angular momentum tracking
- Barnes–Hut statistics (node count, depth, traversal cost)
- Optional octree debug visualization
- ImGui panels for physics and engine inspection

Known Limitations
- Diagnostics add runtime overhead
- Octree caching and pooling are experimental

-----------------------------
v0.5.0 – ImGui & Runtime Tuning
-----------------------------

Changes
- Integrated Dear ImGui via C++ layer
- Live editing of gravity, time scale, pause state
- Renderer split into world and GUI passes
- Introduced RenderContext abstraction
- Improved planet shader and basic LOD

Known Limitations
- GUI is editor-only
- Runtime edits are not persisted

---------------------------------------------
v0.4.0 – Global Configuration System
---------------------------------------------

Changes
- Centralized EngineConfig accessed via config()
- Removed config plumbing between systems
- Runtime clamping and validation of values
- Safer defaults for physics parameters

Known Limitations
- No runtime hot-reload of configuration

-----------------------------------------
v0.3.0 – Stable Orbits & Camera Unification
-----------------------------------------

Changes
- Unified camera system (free, orbit, follow)
- Symplectic (Velocity Verlet) integration
- Fixed Barnes–Hut traversal errors
- Improved gravitational softening
- Deterministic orbital initialization
- Optional per-body damping

Known Limitations
- Barnes–Hut rebuilt every simulation step
- No collision handling
- No floating-origin support

--------------------------------
v0.2.0 – Scalable Gravity
--------------------------------

Changes
- Replaced N² gravity with Barnes–Hut octree
- Configurable theta and softening parameters
- Automatic initial velocity assignment
- Deterministic fixed-timestep physics

Known Limitations
- Not energy-conserving over long runs
- No octree caching or pooling

-------------------------
v0.1.0 – Initial Engine
-------------------------

Changes
- Fixed-timestep scheduler
- World container with stable IDs
- Basic N-body gravity
- Orbit-based camera system
- Shader-based planet rendering
- Centralized logging and config loading

Known Limitations
- O(n²) gravity
- No collisions
- Limited stability at extreme time scales
