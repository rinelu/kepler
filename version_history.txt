Kepler – Revision History
================================
Semantic versioning: MAJOR.MINOR.PATCH

------------------------------------------------
v0.8.0 – Rendering, Camera, and Physics Enhancements
------------------------------------------------

Changes
- Full rewrite of celestial rendering using BodyRender abstraction
- Added bloom post-processing with HDR extraction, blur, and combine passes
- Added glow/rim shaders for emissive stars
- Multi-LOD sphere meshes for planets (near, mid, far)
- Unified camera system:
    - Free, orbit, and follow modes fully integrated
    - Smooth interpolation for position, yaw, pitch, and distance
    - Mouse axis inversion configurable via GUI
- Barnes–Hut gravity improvements and bug fixes (`bh_compute_gravity`)
- Adaptive physics timestep scaling with sub-stepping for high-acceleration bodies
- Fixed planet shader normals
- Optimized render data and body creation pipeline

Known Limitations
- Barnes–Hut still violates Newton’s 3rd law
- No collision or body merging support
- Floating-origin support not yet implemented
- Some visual effects (atmospheres) are incomplete

------------------------------------------------
v0.7.0 – Lighting, Orbits, and Physics Stability
------------------------------------------------

Changes
- Multi-light point lighting with distance attenuation
- Blackbody temperature → RGB for star coloration
- Emissive stars and atmospheric rim lighting
- Explicit orbital initializers (Keplerian, virial rotating)
- Adaptive physics sub-stepping based on max acceleration
- Automatic switch between N² gravity and Barnes–Hut
- Simplified body creation and render data cleanup

Known Limitations
- Barnes–Hut violates Newton’s 3rd law
- No collisions or body merging
- No floating-origin support

--------------------------------
v0.6.0 – Diagnostics & Introspection
--------------------------------

Changes
- Total energy (K + U) tracking per fixed step
- Linear and angular momentum tracking
- Barnes–Hut statistics (node count, depth, traversal cost)
- Optional octree debug visualization
- ImGui panels for physics and engine inspection

Known Limitations
- Diagnostics add runtime overhead
- Octree caching and pooling are experimental

-----------------------------
v0.5.0 – ImGui & Runtime Tuning
-----------------------------

Changes
- Integrated Dear ImGui via C++ layer
- Live editing of gravity, time scale, pause state
- Renderer split into world and GUI passes
- Introduced RenderContext abstraction
- Improved planet shader and basic LOD

Known Limitations
- GUI is editor-only
- Runtime edits are not persisted

---------------------------------------------
v0.4.0 – Global Configuration System
---------------------------------------------

Changes
- Centralized EngineConfig accessed via config()
- Removed config plumbing between systems
- Runtime clamping and validation of values
- Safer defaults for physics parameters

Known Limitations
- No runtime hot-reload of configuration

-----------------------------------------
v0.3.0 – Stable Orbits & Camera Unification
-----------------------------------------

Changes
- Unified camera system (free, orbit, follow)
- Symplectic (Velocity Verlet) integration
- Fixed Barnes–Hut traversal errors
- Improved gravitational softening
- Deterministic orbital initialization
- Optional per-body damping

Known Limitations
- Barnes–Hut rebuilt every simulation step
- No collision handling
- No floating-origin support

--------------------------------
v0.2.0 – Scalable Gravity
--------------------------------

Changes
- Replaced N² gravity with Barnes–Hut octree
- Configurable theta and softening parameters
- Automatic initial velocity assignment
- Deterministic fixed-timestep physics

Known Limitations
- Not energy-conserving over long runs
- No octree caching or pooling

-------------------------
v0.1.0 – Initial Engine
-------------------------

Changes
- Fixed-timestep scheduler
- World container with stable IDs
- Basic N-body gravity
- Orbit-based camera system
- Shader-based planet rendering
- Centralized logging and config loading

Known Limitations
- O(n²) gravity
- No collisions
- Limited stability at extreme time scales
